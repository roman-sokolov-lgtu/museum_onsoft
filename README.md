# Музейный ИИ-ассистент (RAG-прототип)

Минимальный прототип виртуального гида музея на основе **локальной LLM** (Ollama) с RAG-поиском по текстам экспонатов. Отвечает только на основе загруженных текстов и всегда указывает источник.

## Быстрый старт

### Шаг 0 — Предварительно (один раз)

Установить [Ollama](https://ollama.com) и скачать модели:

```bash
ollama pull qwen2.5:3b
ollama pull nomic-embed-text
```

> После установки Ollama запускается автоматически как фоновый сервис.

---

### Шаг 1 — Установить зависимости

```bash
pip install -r requirements.txt
```

### Шаг 2 — Проиндексировать тексты экспонатов

```bash
python index.py
```

Скрипт прочитает все `.txt` файлы из папки `content/`, создаст эмбеддинги через Ollama и сохранит в локальную базу ChromaDB. Достаточно запустить **один раз** (или после добавления новых текстов).

### Шаг 3 — Запустить сервер

```bash
uvicorn main:app --reload
```

Открыть в браузере: **http://localhost:8000**

## Возможности и фишки

- **Умный фильтр галлюцинаций:** система строго сверяет ответы нейросети с текстом (по корням слов), пресекая выдуманные факты.
- **Мгновенный каталог:** система распознаёт запросы типа «полный список», «каталог» или «какие экспонаты» и мгновенно возвращает красивый перечень всех произведений в обход ресурсоёмкой LLM.
- **Поддержка контекста:** бот учитывает ваш предыдущий вопрос при поиске, позволяя вести диалог (например: "какие тут скульптуры?" -> "почему тройка?").

## Как это работает (RAG-цепочка)

```
Вопрос посетителя
       │
       ▼
[Ollama: nomic-embed-text]  ←─ эмбеддинг вопроса
       │
       ▼
[ChromaDB: поиск топ-3]     ←─ семантический поиск по базе
       │
       ▼
[Промпт = вопрос + контекст]
       │
       ▼
[Ollama: qwen2.5:3b]        ←─ генерация ответа
       │
       ▼
Ответ + список источников (имена .txt файлов)
```

## Технологии

| Компонент | Технология |
|-----------|-----------|
| LLM | Ollama + `qwen2.5:3b` |
| Эмбеддинги | Ollama + `nomic-embed-text` |
| Векторная БД | ChromaDB (локальная) |
| Бэкенд | Python + FastAPI |
| Фронтенд | HTML + Vanilla JS |

## Добавление своих экспонатов

1. Создайте `.txt` файл в папке `content/` с описанием экспоната
2. Запустите `python index.py` заново
3. Перезапустите сервер
